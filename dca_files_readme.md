# The Efficient Frontier of Dollar Cost Averaging — pipeline reference

Everything needed to build the data, run the tests, and produce `wealth_report.pdf`.

## Run it

```
python3 make_report.py             # full run: data -> Monte Carlo -> tables/stats/figures -> PDF
python3 make_report.py --no-fetch  # skip the data download, reuse the committed CSVs
python3 make_report.py --skip-mc   # skip the Monte Carlo, reuse the existing results/ stores
python3 make_report.py --seed-check # also run the second-seed (seed 7) robustness check, see below
```

Requirements: Python 3 with `numpy`, `pandas`, `scipy`, `matplotlib`; a TeX install with `pdflatex`;
network access (Yahoo Finance, the LBMA gold benchmark) for the data stage; and `reit_monthly.csv`
present (a one-time manual download, see below). The Monte Carlo stage takes roughly 40 minutes.

## Pipeline stages

1. **Data** — build the single 1980–2026 dataset (eight legs: US stocks, tech, international, gold,
   intermediate bonds, long Treasuries, cash, REITs).
2. **Monte Carlo** — 1,000 block-bootstrap histories per strategy, both tax regimes.
3. **Tables / stats / figures** — the full-field table, the distribution stats, and the figures.
4. **Compile** — `wealth_report.tex` → `wealth_report.pdf`.

## Files

**Orchestrator**
- `make_report.py` — runs all stages in order and compiles the report.

**Data build**
- `dca_build80.py` — fetches the five fund legs (US stocks, tech, international, gold, bonds) back to
  1980 and writes the base dataset; `dca_build80.py merge` joins the constructed legs (below) into the
  final 8-leg files `dca_adj_div_80full.csv` / `dca_yield_div_80full.csv`.
- `build_rate_legs.py` — constructs the **long-Treasury** and **cash (T-bill)** legs from the
  constant-maturity Treasury yields, net of the matching fund's expense ratio, spliced into the real
  funds (VUSTX 1986, VFISX 1991). Writes `longt_series.csv`, `cash_series.csv`.
- `build_reit.py` — constructs the **REIT** leg from the Nareit monthly index (pre-1986) spliced into
  the FRESX→VGSIX fund chain. Writes `reit_series.csv`. Reads `reit_monthly.csv`.
- `reit_monthly.csv` — input: FTSE Nareit U.S. Real Estate Index Series monthly returns, downloaded
  from `https://www.reit.com/sites/default/files/returns/MonthlyHistoricalReturns.xls`.

**Engine**
- `dca_core.py` — the simulation engine: assets, strategies, the tax/fee model (0.4% gold management
  fee, 0.1% transaction cost, income and capital-gains tax), the block bootstrap, and the runners
  (static no-reb / soft / hard, age glides, momentum, signal overlays).
- `div_rebin.py` — the received-preserving dividend-cadence re-bin used by the bootstrap.

**Monte Carlo**
- `parallel_recompute.py` — runs the full field over 1,000 paths for both tax regimes across CPU cores;
  writes `results/tpb_1980_{taxable,taxfree}.npy` and `results/core_1980_{taxable,taxfree}.npy`.
  `python3 parallel_recompute.py 1980 <seed>` writes seed-tagged stores (`_s<seed>`) for a robustness
  re-run; the default seed-42 stores are never overwritten.

**Tables, stats, figures**
- `gen_tables2.py` — the per-strategy Appendix table → `wealth_tables.tex` (pulled into the report).
- `compute_stats.py` — per-strategy distribution stats and paired tests → `results_record.md`.
- `plot_frontier.py` — the wealth-vs-terminal-pain frontier diagrams → `results/frontier_1980_*.png`;
  also prints the within-cap efficient ranking (`stability` and `s<seed>` modes for robustness).
- `calendar_compare.py` — calendar-year returns (60/40 vs diversified 30/30/40) and asset-class worst
  drawdowns → `results/calendar_60_40_vs_div.png`, `calendar_rows_*.tex`, `downyears_rows.tex`,
  `asset_dd_rows.tex`.
- `robustness.py` — the income-path and lazy-investor (skip-25%-of-contributions) checks →
  `results/income_rows.tex`, `lazy_rows.tex`.
- `horizon_robust.py` — re-evaluates the within-cap efficient set at shorter accumulation horizons
  (31–46 years); static mixes are sliced from one path, glides rebuilt to complete at each horizon →
  `results/horizon_rows.tex`, `horizon_div_rows.tex` (cache: `results/horizon_raw.npy`).

**Report**
- `wealth_report.tex` → `wealth_report.pdf`. Inputs `wealth_tables.tex` and the figures above.

## Data files produced

- `dca_adj_div_80full.csv` / `dca_yield_div_80full.csv` — the unified 8-leg 1980–2026 dataset (price
  level + income yield).
- `longt_series.csv`, `cash_series.csv`, `reit_series.csv` — the three constructed legs.
- `results/tpb_1980_*.npy`, `results/core_1980_*.npy` — the Monte Carlo stores.
- `results/frontier_1980_*.png`, `results/calendar_60_40_vs_div.png` — the figures.
- `wealth_tables.tex`, `results_record.md` — the generated table and stats record.

## Reproducibility

Every table and figure in the report is generated by the stages above and pulled into the PDF via
`\input`/`\includegraphics` — the full-field table (`wealth_tables.tex`), the efficient-frontier sets
(`results/eff_*.tex`), the calendar-year and down-year tables, the asset-drawdown table, the income-path,
lazy-investor, and horizon tables, and all figures. Nothing in the report is hand-transcribed;
re-running the pipeline regenerates every number.

## Second-seed robustness check

The report rests its conclusions on the strategies that stay on the efficient frontier under a second,
independent random seed (rather than any single strategy's rank at the fixed seed). That check is a
separate ~40-minute Monte Carlo, so it is **not** part of the default run. Reproduce it with:

```
python3 make_report.py --seed-check
```

which runs `parallel_recompute.py 1980 7` (writing the seed-tagged `results/{tpb,core}_1980_*_s7.npy`
stores) and then `plot_frontier.py 1980 {taxable,taxfree} s7`, printing the seed-7 within-cap efficient
sets to compare against the committed seed-42 sets in `results/eff_taxable.tex` / `eff_taxfree.tex`.
`plot_frontier.py 1980 <regime> stability` is a faster proxy: it path-resamples the seed-42 stores and
reports how often each strategy lands on the frontier (high % = robust, not seed-luck). Note the `.npy`
stores are gitignored and regenerated by this step; the committed `results/frontier_1980_*_s7.png`
figures are the persisted seed-7 frontier diagrams.
